# Экспериментальные функции 

---

Раздел предназначен для включения и настройки функций, находящихся на стадии тестирования или предназначенных для узкоспециализированных сценариев использования. Параметры, собранные здесь, могут быть изменены без обновления сервера, однако их некорректная конфигурация способна повлиять на стабильность, безопасность и производительность системы. Рекомендуется использовать данный раздел только опытным администраторам или по согласованию со службой технической поддержки DST App.

Раздел состоит из двух подразделов:
- **Экспериментальные функции** — настройки, влияющие на поведение клиентских приложений, производительность, внешний вид элементов аутентификации и пользовательский опыт.
- **Флаги функций** — системные переключатели, включающие или отключающие отдельные функциональные блоки (например, управление доступом на основе атрибутов, маркировку контента). Изменение флагов функций без рекомендаций разработчика может привести к непредсказуемой работе системы.

---

## Экспериментальные функции

**Что это и зачем**  
Набор параметров, позволяющих адаптировать поведение DST App под специфику инфраструктуры, требования безопасности или предпочтения пользователей. Некоторые функции находятся в стадии тестирования и могут быть изменены или удалены в будущих версиях. Данный раздел предназначен для тонкой настройки, которая не вошла в основные разделы панели управления.

**Основные группы параметров**

### 1. Настройки внешнего вида кнопок аутентификации (мобильные приложения)

Для методов входа AD/LDAP, электронная почта и SAML можно задать индивидуальные цвета фона, границы и текста кнопки (шестнадцатеричный код, например `#FF6600`). Это позволяет визуально различать способы входа на экране авторизации мобильного приложения, что особенно полезно при одновременном использовании нескольких методов аутентификации.

- **Рекомендация:** Используйте корпоративные цвета бренда для основного метода входа (например, электронная почта), а для альтернативных (AD/LDAP, SAML) — нейтральные оттенки. Не применяйте более трех различных цветов, чтобы не перегружать интерфейс.

### 2. Управление аутентификацией и учетными записями

- **Разрешить передачу аутентификации** (Да/Нет). При значении «Да» пользователи могут самостоятельно сменить метод входа (например, с пароля на SSO) через настройки учетной записи или API. При значении «Нет» метод входа фиксирован независимо от того, какие способы аутентификации включены на сервере.
  - *Рекомендация:* В организациях с жесткой политикой безопасности (госорганы, финансы) установите «Нет», чтобы исключить возможность обхода корпоративного SSO.

- **Включить деактивацию аккаунта** (Да/Нет). Разрешает пользователям самостоятельно деактивировать свою учетную запись через раздел «Параметры → Дополнительно». При деактивации пользователь получает подтверждение по электронной почте.
  - *Рекомендация:* Для большинства корпоративных сред установите «Нет», чтобы управление учетными записями оставалось централизованным (через AD/LDAP или администратора).

- **Включить автоматические ответы** (Да/Нет). Позволяет пользователям настраивать автоматические ответы на личные сообщения (например, в отпуске). При значении «Нет» функция скрывается из интерфейса.
  - *Рекомендация:* Включите для компаний с высокой динамикой коммуникаций (поддержка клиентов, диспетчерские). Отключите в организациях, где автоматические ответы могут ввести в заблуждение (например, операционные центры).

### 3. Производительность и сетевые настройки

- **Включить WebSocket-сообщения «Channel Viewed»** (Да/Нет). Определяет, будут ли отправляться события о просмотре канала для синхронизации непрочитанных уведомлений между устройствами. Отключение снижает нагрузку на сервер в крупных развертываниях (более 5 000 пользователей).
  - *Рекомендация:* Для компаний с численностью менее 2 000 пользователей оставьте «Да». Для крупных организаций протестируйте с «Нет» и оцените влияние на синхронизацию.

- **Включить сообщения «пользователь печатает...»** (Да/Нет) и **Тайм-аут ввода пользователя** (миллисекунды). Отображение индикатора набора текста повышает удобство, но генерирует дополнительный трафик. Отключение может повысить производительность при очень высокой нагрузке.
  - *Рекомендация:* Оставляйте включенным для большинства сценариев. Увеличьте тайм-аут до 5–7 секунд, чтобы снизить частоту отправки событий.

- **Интервал опроса состояния и профиля пользователя** (миллисекунды). Частота обновления статусов (онлайн/офлайн) и профилей. Более редкий опрос снижает нагрузку на базу данных, но статусы обновляются медленнее.
  - *Рекомендация:* Значение по умолчанию — 60 000 мс (1 минута). Для критичных к оперативности служб (диспетчерские) можно уменьшить до 30 000 мс, для крупных организаций — увеличить до 120 000 мс.

- **Отключить повторное получение данных при перефокусировке браузера** (Да/Нет). При значении «Да» система не перезапрашивает каналы и участников при возврате фокуса на вкладку браузера. Это снижает трафик для пользователей с большим количеством каналов.
  - *Рекомендация:* Включите для организаций, где сотрудники работают с десятками каналов и часто переключаются между приложениями.

### 4. Поведение каналов и сообщений

- **Включение системных сообщений «покинул/присоединился» в канале по умолчанию** (Да/Нет). Публикует уведомления о входе/выходе пользователей в основной канал команды (Town Square). Отключение снижает информационный шум в больших командах.
  - *Рекомендация:* Для команд численностью более 100 человек установите «Нет».

- **Тайм-аут метаданных ссылки** (миллисекунды). Время ожидания получения метаданных (заголовок, описание, превью) при вставке внешней ссылки. Слишком короткий тайм-аут может привести к отсутствию превью, слишком длинный — замедлит отправку сообщения.
  - *Рекомендация:* Оставьте значение по умолчанию (обычно 5 000 мс). Увеличьте до 10 000 мс для ссылок на медленные внутренние ресурсы.

- **Задержка автозаполнения канала** (Да/Нет). При значении «Да» автозаполнение ссылок на каналы (через `~название`) срабатывает только после ввода тильды и нескольких букв. При «Нет» — сразу после ввода тильды.
  - *Рекомендация:* Включите для серверов с большим количеством каналов (более 500), чтобы снизить нагрузку на клиентскую часть.

### 5. Пользовательский интерфейс и темы

- **Включить выбор темы** (Да/Нет) и **Разрешить пользовательские темы** (Да/Нет). Позволяют пользователям выбирать светлую/темную тему или создавать свои. **Тема по умолчанию** задает тему для новых пользователей.
  - *Рекомендация:* Включите выбор темы для повышения комфорта. Пользовательские темы отключайте в организациях с требованиями к унификации интерфейса (например, контакт-центры).

- **Включить учебник** (Да/Нет) и **Включить адаптацию** (Да/Нет). При первом входе пользователю показывается обучающий тур или шаги адаптации. Помогает новым сотрудникам быстрее освоиться.
  - *Рекомендация:* Включайте для компаний с высокой текучестью или регулярным наймом. Отключайте для небольших стабильных команд.

### 6. Безопасность и комплаенс

- **Включить усиленный режим** (Да/Нет). Режим, который жертвует удобством (например, отключает сохранение паролей в браузере, запрещает простые ссылки) в пользу повышенной безопасности. Рекомендуется для организаций, работающих с гостайной или персональными данными высокого риска.
  - *Рекомендация:* Включайте только после согласования со службой ИБ и пилотного тестирования на ограниченной группе.

### 7. Прочие настройки

- **Основная команда** (идентификатор команды). Пользователи автоматически становятся членами этой команды и не могут ее покинуть. Также отключается возможность создавать новые команды.
  - *Рекомендация:* Используйте для создания единого корпоративного пространства, где все сотрудники обязаны состоять (например, «Вся компания»). Для проектных групп создавайте отдельные команды без статуса «основная».

- **Тайм-аут статуса пользователя «Отошёл»** (секунды). Время неактивности, после которого статус пользователя меняется с «Онлайн» на «Отошёл».
  - *Рекомендация:* Для офисных сотрудников — 600 секунд (10 минут). Для диспетчеров или служб поддержки — 120 секунд (2 минуты).

- **Политика рефереров YouTube** (Да/Нет). Влияет на отображение превью видео с YouTube. «Да» — превью работают, но передается минимальная информация о источнике. «Нет» — превью могут не отображаться, но повышается конфиденциальность.
  - *Рекомендация:* Установите «Да», если сотрудники часто обмениваются видео. «Нет» — для организаций с жесткой политикой защиты от утечек данных.

- **Сортировка категорий каналов** (Да/Нет). При значении «Да» каналы автоматически сортируются по категориям на основе имени с разделителем «/» (например, «Проект/Дизайн», «Проект/Разработка» группируются в категорию «Проект»).
  - *Рекомендация:* Включите для серверов с иерархической структурой каналов (более 100 каналов).

> **Важное предупреждение:** Изменение параметров в разделе «Экспериментальные функции» без понимания их влияния на систему может привести к нестабильной работе, снижению производительности или нарушению пользовательского опыта. Перед внесением изменений в production-среду протестируйте их на отдельном тестовом сервере.

## Флаги функций

**Что это и зачем**  
Флаги функций (feature flags) — это внутренние переключатели, которые включают или отключают отдельные функциональные блоки в коде DST App. Они используются разработчиками для поэтапного внедрения новых возможностей, A/B-тестирования и быстрого отключения функции в случае обнаружения проблем. В обычной эксплуатации изменение флагов функций не требуется и может привести к некорректной работе системы.

**Основное назначение для администратора**  
- Диагностика проблем по запросу службы поддержки DST App.  
- Временное включение/отключение функции для проверки гипотезы (только на тестовом сервере).  
- Получение информации о том, какие функции активны в текущей версии.

**Как использовать**  
Раздел отображает текущее состояние флагов (true — включено, false — отключено). Изменять значения следует **только по указанию технической поддержки DST App** или в рамках отладочных работ на изолированном стенде. Самостоятельное изменение флагов на production-сервере может привести к потере данных, сбоям в работе и невозможности обновления системы.

**Наиболее значимые флаги (для понимания)**  

| Флаг | Описание | Рекомендация |
|------|----------|---------------|
| `AttributeBasedAccessControl` | Включает управление доступом на основе атрибутов (ABAC). Если вы используете раздел «Системные атрибуты», этот флаг должен быть `true`. | Не изменяйте. Если ABAC не работает, обратитесь в поддержку. |
| `ContentFlagging` | Включает маркировку контента (возможность помечать сообщения как «конфиденциально», «для внутреннего использования»). | Оставьте `true`, если используете соответствующую функцию в настройках сайта. |
| `CustomProfileAttributes` | Позволяет добавлять пользовательские поля в профиль пользователя. | `true` — рекомендуется для большинства корпоративных развертываний. |
| `NotificationMonitoring` | Включает мониторинг доставляемости уведомлений (логи, метрики). Полезно для диагностики проблем с push/email. | `true` — оставляйте включенным. |
| `BurnOnRead` | Реализует режим «самоуничтожающихся» сообщений (прочтено — удалено). | Включайте только если есть явный бизнес-сценарий (передача паролей, секретов). |
| `MobileSSOCodeExchange` | Обеспечивает обмен SSO-кодов в мобильных приложениях. | Должен быть `true` для корректной работы единого входа на iOS/Android. |
| `StreamlinedMarketplace` | Включает упрощенный интерфейс маркетплейса плагинов. | `true` — рекомендуется. |
| `WysiwygEditor` | Включает визуальный редактор сообщений (WYSIWYG). Если `false`, используется редактор Markdown. | Настраивайте в зависимости от предпочтений пользователей. `true` — проще для не-IT-сотрудников. |

**Флаги, которые не следует изменять без указания поддержки:**  
`AppsEnabled`, `CloudAnnualRenewals`, `EnableRemoteClusterService`, `EnableSharedChannelsDMs`, `ExperimentalAuditSettingsSystemConsoleUI`, `MoveThreadsEnabled`, `PermalinkPreviews` и другие, не описанные в таблице выше. Их изменение может нарушить лицензионную политику, кластеризацию или синхронизацию между серверами.

> **Важное предупреждение:** Раздел «Флаги функций» предназначен **только для отладки командой поддержки DST App**. Несанкционированное изменение флагов может привести к аннулированию гарантийных обязательств и невозможности обновления системы. Если вы не уверены в назначении флага — не меняйте его.
